Section 02

Portal Intelligence Summary

What we know about portal.growthiq.co.za from the May 2026 authenticated audit. This intelligence directly informs what we build in Phase 4 and what we choose to feature in website copy.

Critical constraint for Phase 4 copywriting Do not feature portal capabilities that are currently broken in website copy. The document expiry alert system is non-functional. The dashboard is showing incorrect data. The mobile experience is broken. Only feature confirmed-working modules. See below.

SME Management (10-tab company records)

  • Company Details, Funding, Employees
  • Documents, Compliance Action List
  • Client Information, Assessments
  • Features, Financials, Operational Progress
  • Comprehensive — supports actual ESD workflow

Compliance Task Tracking

  • Action List per SME: owner, deadline, status
  • Statuses: Overdue / In Progress / Completed / Not Started
  • Multiple overdue tasks confirmed active — system works
  • Problem is navigation to it (4 clicks deep), not functionality

Assessment & Scoring System

  • Scored questionnaires with weighted answers
  • Categories and multiple-choice responses
  • "Generate SME Journey Report" function exists
  • Meaningful differentiator for corporate pitch

Audit Trail

  • Every user action logged with timestamps
  • Compliance-grade paper trail
  • Critical for corporate reporting accountability
  • Feature this prominently in website copy

Document Expiry Alerts

  • BEE certificate expired Nov 2025 — not flagged
  • Dashboard shows zero upcoming expirations
  • Do NOT feature in Phase 4 website copy
  • Fix required before AI layer (Phase 5 scope)

Error Monitoring (Sentry)

  • Both Sentry files returning errors — monitoring is blind
  • Technical problems going unreported since handover
  • No active NetGen maintenance confirmed
  • Note in Phase 4: our site will have monitoring from day 1

Default Landing Screen

  • Every login lands on raw admin user table
  • Dashboard is a manual navigation away
  • Configuration fix, not a rebuild
  • Phase 4 website must not promise "seamless client experience"

Analytics / Usage Tracking

  • Zero tracking inside portal
  • Cannot confirm which clients are logging in
  • Cannot identify at-risk accounts
  • Portal remediation (later phase) must address this first

Portal workflow — how it actually functions (for Phase 4 "How It Works" page):

Corporate client
→
Engages Growth Firm to manage ESD programme against B-BBEE Amended Codes 2023
SMEs onboarded
→
Each SME gets a full 10-tab record: company profile, documents, funding, employees, compliance tasks
Compliance tasks
→
Action List per SME tracks each obligation: owner assigned, deadline set, status updated in real time
Assessments
→
Scored questionnaires measure SME compliance maturity across weighted categories
Reporting
→
"Generate SME Journey Report" produces compliance progress output for corporate review
Audit trail
→
Every action timestamped — corporate clients can verify compliance activity at any point
Phase 5 CTA vision — AI layer as front door, not the portal directly The Phase 4 marketing site does NOT link users to the raw portal. Instead, the "Access GrowthIQ" CTA leads to the Phase 5 AI assistant interface (e.g. app.growthiq.co.za). The AI greets the user: "How can I help you today?" — and routes them to the right data or walks them through input step by step. This removes the 4-clicks-deep navigation problem entirely. The portal continues to exist as the data layer; the AI is the face. This vision should be hinted at in Phase 4 website copy ("AI-powered compliance management") without promising functionality that isn't live yet. Phase 5 builds it.

Phase 04 · Weeks 5–8

Marketing Website — growthiq.co.za

5-page credibility site. Goal: get corporate prospects to take the call. Not self-serve. Not lead gen. Next.js on Vercel, Cloudflare DNS. SA-market-optimised: mobile-first, WhatsApp CTA, fast on mobile data.

SA market constraint: mobile-first is non-negotiable The corporate target list is SA enterprise, but the decision-makers (Transformation Managers, Procurement) are not desk-bound. Expect the site to be checked on mobile during a meeting or commute. Mobile performance is weighted the same as desktop — not an afterthought. Next.js + Cloudflare CDN handles this at the infrastructure level. Images served in WebP via Next.js Image component. No uncompressed assets.
